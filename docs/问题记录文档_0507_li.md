## 问题详情

### 问题 1：Windows 文件路径与 file:// URL 兼容性问题

**错误信息：**
```
playwright._impl._errors.TimeoutError: Page.goto: Timeout 30000ms exceeded.
Call log:
  - navigating to "file:///C:/Users/FENGWE~1/AppData/Local/Temp/tmpkfomji_n.html", waiting until "load"
```

**原因分析：**
- 脚本使用 `file://{temp_html_path}` 方式构建本地文件 URL
- Windows 路径使用反斜杠 `\`，而 URL 需要正斜杠 `/`
- 临时文件路径如 `C:\Users\...\tmp.html` 未转换导致 URL 格式错误

**解决方案：**
修改 `render_xhs.py` 第 454-458 行，添加路径转换逻辑：
```python
# Convert Windows path to file:// URL format
if os.name == 'nt':
    temp_html_path = temp_html_path.replace('\\', '/')
await page.goto(f'file:///{temp_html_path}')
```

**后续优化：**
最终改为使用 `page.set_content(html_content)` 直接设置内容，避免文件路径问题。

### 问题 2：Playwright set_content/goto 超时问题

**错误信息：**
```
playwright._impl._errors.TimeoutError: Page.set_content: Timeout 30000ms exceeded.
Call log:
  - setting frame content, waiting until "load"
```

**原因分析：**
- Playwright 默认等待 `load` 事件，但某些 HTML 内容可能触发资源加载
- 本地 HTML 中引用的外部资源（字体、CSS）无法访问时导致超时
- `wait_until='networkidle'` 在本地文件场景下不适用

**解决方案：**
1. 移除 `wait_until` 参数，使用默认行为
2. 改用 `page.wait_for_timeout(500)` 等待短时间
3. 最终使用简化版渲染脚本，避免复杂依赖

**修复代码：**
```python
# 原代码
await page.set_content(html_content, wait_until='load')

# 修改后
await page.set_content(html_content)
await page.wait_for_timeout(500)
```

**备用方案：**
创建简化版渲染脚本 `render_simple.py`，使用内联 CSS，不依赖外部资源。

### 问题 3：脚本缩进和 finally 块语法错误

**错误信息：**
```
File "render_xhs.py", line 518
    finally:
    ^^^^^^^
SyntaxError: invalid syntax
```

**原因分析：**
- 修复路径问题时多次编辑导致缩进混乱
- `finally` 块与 `try` 块不匹配
- Python 对缩进敏感，混合使用不同层级导致语法错误

**解决方案：**
1. 检查 `try-finally` 结构完整性
2. 确保 `finally` 与对应 `try` 缩进层级一致
3. 移除不必要的 `finally` 块（因不再需要清理临时文件）

**修复命令：**
```python
# 使用 Python 脚本批量修复缩进
with open('render_xhs.py', 'r', encoding='utf-8') as f:
    content = f.read()
content = content.replace('        finally:\n        await browser.close()', 
                          '    finally:\n        await browser.close()')
with open('render_xhs.py', 'w', encoding='utf-8') as f:
    f.write(content)
```

### 问题 5：Cookie 获取与配置流程

**错误信息：**
```
❌ 错误：未找到 XHS_COOKIE 环境变量
请创建 .env 文件，添加以下内容：
XHS_COOKIE=your_cookie_string_here
```

**原因分析：**
- 小红书发布需要用户认证
- Cookie 包含登录状态（`a1`、`web_session` 等关键字段）
- 需要手动从浏览器获取并配置

**解决方案流程：**

1. **创建 Cookie 获取脚本** (`quick_login.py`)：
```python
from playwright.async_api import async_playwright
import asyncio

async def main():
    p = await async_playwright().start()
    browser = await p.chromium.launch(headless=False)
    context = await browser.new_context()
    page = await context.new_page()
    await page.goto("https://www.xiaohongshu.com")
    print("等待 60 秒登录...")
    await asyncio.sleep(60)
    cookies = await context.cookies()
    cookie_string = "; ".join([f"{c['name']}={c['value']}" for c in cookies])
    with open('.env', 'w', encoding='utf-8') as f:
        f.write(f'XHS_COOKIE={cookie_string}\n')
    await browser.close()
    await p.stop()

asyncio.run(main())
```

2. **运行脚本获取 Cookie：**
```bash
cd C:\Users\fengweione\.agents\skills\xhs-note-creator
python scripts/quick_login.py
```

3. **验证 Cookie 有效性：**
- 必须包含 `a1` 字段
- 必须包含 `web_session` 字段

**手动获取方法（备用）：**
1. 浏览器访问 https://www.xiaohongshu.com 并登录
2. 按 F12 打开开发者工具
3. Network 标签 → 刷新页面 → 点击任意请求
4. 复制 Request Headers 中的 Cookie 值
5. 保存到 `.env` 文件：`XHS_COOKIE=你的 cookie`
